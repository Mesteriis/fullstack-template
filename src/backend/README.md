# Backend

FastAPI backend baseline for the fullstack template.

## What Lives Here

- bounded contexts under `apps/`
- shared platform/runtime code under `core/` and `runtime/`
- Alembic migrations in the repository-root `migrations/`
- backend-local tests in `tests/`

## Commands

```bash
make help
make sync
make lint
make types
make test
make test-migrations
make migrate
make makemigration m="describe change"
make current
make history
make compose-up
make compose-down
```

## Migration Testing

Backend tests include `pytest-alembic`.

- migration tests run against a PostgreSQL database created inside a `testcontainers` container;
- a fresh database is created for the Alembic test run and dropped afterwards;
- live local databases are not used for migration verification.

## Migration Commands

- `make migrate`: apply all migrations to the configured database
- `make downgrade rev=<revision>`: downgrade to a specific revision
- `make makemigration m="message"`: create a new autogenerated migration

Use the repository-root `.env` for runtime configuration. Backend settings read
their own namespaces only: `APP__*`, `API__*`, `DB__*`, `BROKER__*`,
`SYSTEM__*`, `OBSERVABILITY__*`.

Note: the backend intentionally targets Python 3.14. Syntax that is valid for
Python 3.14 is part of the template baseline and should not be downgraded to
fit older interpreters.
